# 通用公式与病例约束在线微调方案

## 1. 背景与目标
当前系统需要同时满足两类要求：
- 物理一致性：玩家输入参数应通过同一套机制映射到可解释的光斑强度/等级。
- 病例适配性：不同病例存在个体差异，不能完全依赖单一固定阈值。

本方案采用“通用公式 + 病例约束在线微调”架构：
- 离线阶段：确定全局共享的通用公式与等级映射。
- 在线阶段：读取当前病例约束，仅对病例校正项做小幅调整，不改动全局公式参数。

## 2. 总体思路
### 2.1 通用公式（全局共享）
定义通用强度分数：

s = f(P, S, T, λ, Δz; θ)

其中：
- P：功率
- S：光斑直径
- T：曝光时间
- λ：波长
- θ：全局参数（离线训练后固定）

### 2.2 通用映射（全局共享）
将强度分数 s 映射为光斑等级 g（1-4级）：
- g = 1, 若 s < τ1
- g = 2, 若 τ1 <= s < τ2
- g = 3, 若 τ2 <= s < τ3
- g = 4, 若 s >= τ3

其中 τ1, τ2, τ3 为全局阈值。

### 2.3 病例在线微调（仅病例校正项）
对于当前病例 c，引入小尺度校正：

s' = a_c * s + b_c

其中：
- a_c：病例尺度校正（通常接近 1）
- b_c：病例偏移校正（通常接近 0）

最终等级由 s' 通过同一组全局阈值映射得到。

## 3. 为什么在线微调可行
在线微调不是“在线重训练全模型”，而是“在线求解小参数”：
- 不改 θ（通用公式参数）
- 只根据病例约束快速估计 a_c, b_c

这样能兼顾：
- 统一标准：全系统规则一致
- 个体适配：每个病例可在统一标准下做小幅修正
- 实时性：在线可在毫秒到百毫秒级完成

## 4. 病例约束的输入形式
推荐将病例约束组织为若干条规则，每条规则至少包含：
- 约束输入参数 x_i（对应某组 P,S,T,λ,Δz）
- 目标等级 g_i（或目标等级区间）
- 可选权重 w_i（关键约束可赋更高权重）

示例（概念）：
- 约束1：GT参数应映射到 grade=3
- 约束2：低功率短曝光应不超过 grade=2
- 约束3：高功率长曝光应达到 grade=4

## 5. 在线求解目标
在线对 a_c, b_c 求解可采用以下目标：

最小化：
- 约束误差（预测等级与目标等级不一致）
- 正则项（约束校正幅度不要过大）

可写成概念形式：
- Loss = 约束损失 + λ_reg * [ (a_c - 1)^2 + b_c^2 ]

建议加入硬边界：
- a_c ∈ [0.85, 1.15]
- b_c ∈ [-0.4, 0.4]

保证在线微调不会破坏通用公式。

## 6. 在线运行流程
1. 进入病例：加载该病例约束列表。
2. 计算通用分数：对约束样本计算 s_i = f(x_i; θ)。
3. 在线求解：得到本病例 a_c, b_c。
4. 治疗过程：玩家每次输入先算 s，再算 s' = a_c * s + b_c，再映射等级。
5. 会话结束：记录本病例校正参数与预测结果，用于后续离线分析。

## 7. 与当前系统的关系
- 可保留现有 LaserPhysicalModel 作为通用公式主体。
- 在不改变主接口的前提下，新增病例校正层（a_c, b_c）。
- 渲染层与评分层都可共享 s' 或共享等级结果，保持行为一致。

## 8. 落地注意事项
- 不要在线修改全局阈值 τ1/τ2/τ3，避免系统标准漂移。
- 不要让病例校正过大，避免“每个病例一套公式”的过拟合。
- 约束条目至少包含1条 GT 强约束，最好再加若干边界约束。
- 若约束不足，可回退到 a_c=1, b_c=0（即纯通用公式）。

## 9. 后续实现建议
- 第一步：实现病例校正参数的数据结构与加载。
- 第二步：实现在线求解器（可先用网格搜索或小规模梯度法）。
- 第三步：在 LaserAppManager 中加入病例进入时的校正初始化流程。
- 第四步：增加调试输出（a_c, b_c, 关键约束命中率）。

---
本方案核心原则：
- 通用公式负责“统一物理规则”
- 在线微调负责“病例个体适配”
- 两者分层，保证可解释、可控、可扩展
